# Contributing to AI at Scale: Companion Code

We love your input! We want to make contributing to this project as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features (e.g., enhancements to the calculators)
- Expanding the examples in the chapters

## We Develop with GitHub

We use GitHub to host code, to track issues and feature requests, as well as accept pull requests.

## Pull Requests are Welcome!

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Keep your commits clean and atomic.
4. Ensure your code lints and is formatted according to standard Python practices (e.g., PEP 8).
5. Open a Pull Request!

## Reporting Bugs / Issues

When opening an issue, please ensure:
1. You describe the problem clearly and concisely.
2. You provide the steps necessary to reproduce the issue.
3. You mention the Python version and OS you are using.
